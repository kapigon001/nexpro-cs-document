## 2. 動画制作ワークフロー

### 2.1 全体フロー

```
企画 → 台本作成 → レビュー → 撮影 → 編集 → レビュー → 修正 → 公開
(1日)   (2日)      (1日)    (1日)   (2日)   (1日)   (1日)   (0.5日)
```

**1本あたりの標準工数**: 約9.5営業日（2週間弱）

### 2.2 工程詳細

| 工程 | 所要時間 | 担当 | 成果物 |
|------|---------|------|--------|
| 1. 企画 | 1日 | 鈴木 | 動画構成案 |
| 2. 台本作成 | 2日 | 鈴木 | 台本 |
| 3. 台本レビュー | 1日 | 嘉生 | レビューコメント |
| 4. 撮影 | 1日 | 鈴木 | RAW動画 |
| 5. 編集 | 2日 | 鈴木 | 編集済み動画 |
| 6. 動画レビュー | 1日 | 嘉生 | 修正指示 |
| 7. 修正 | 1日 | 鈴木 | 最終版動画 |
| 8. 公開 | 0.5日 | 鈴木 | YouTube限定公開URL |

### 2.3 週次スケジュール例（2本/週ペース）

```
月曜: 動画A台本作成、動画B企画
火曜: 動画A台本作成、動画B台本作成
水曜: 動画Aレビュー、動画B台本作成
木曜: 動画A撮影、動画Bレビュー
金曜: 動画A編集、動画B撮影
```

## 3. 台本フォーマット

### 3.1 台本に含める要素

| 要素 | 説明 | 必須/任意 |
|------|------|----------|
| 基本情報 | 動画番号、タイトル、対象者、想定時間 | 必須 |
| 学習目標 | 視聴後に何ができるようになるか | 必須 |
| 前提知識 | 事前に視聴すべき動画 | 必須 |
| セクション構成 | 各パートの時間配分と内容 | 必須 |
| ナレーション | 読み上げる文章 | 必須 |
| 画面操作指示 | クリック位置、入力内容、画面遷移 | 必須 |
| テロップ指示 | 表示するテキスト、タイミング | 必須 |
| 強調ポイント | ズームやハイライトする箇所 | 任意 |
| よくある質問 | FAQ | 任意 |

### 3.2 台本テンプレート

```markdown
# 動画[番号] - [タイトル]

## 基本情報

| 項目 | 内容 |
|------|------|
| 動画番号 | No.X |
| タイトル | [タイトル] |
| 対象者 | [全共通 / ライブ / 疑似ライブ / オンデマンド / etc.] |
| 想定時間 | X分 |
| 作成日 | YYYY/MM/DD |
| 最終更新 | YYYY/MM/DD |
| 担当 | 鈴木 |

## 学習目標

この動画を視聴すると、以下ができるようになります：
- [ ] 目標1
- [ ] 目標2
- [ ] 目標3

## 前提知識・事前視聴

- 動画X「[タイトル]」を視聴済みであること
- ネクプロ管理画面にログインできること

| セクション | 時間 | 内容 |
|-----------|------|------|
| 1. オープニング | 0:00-0:30 | 動画の概要・学習目標 |
| 2. [セクション名] | 0:30-2:00 | [内容] |
| 3. [セクション名] | 2:00-4:00 | [内容] |
| 4. まとめ | X:XX-X:XX | ポイント整理・次のステップ |

## 詳細台本

### セクション1: オープニング（0:00-0:30）

| 時間 | ナレーション | 画面操作 | テロップ |

### セクション2: [セクション名]（0:30-2:00）

| 時間 | ナレーション | 画面操作 | テロップ | 強調 |
|------|-------------|---------|---------|------|
| 0:30 | 「まず、[操作]を行います。」 | [メニュー]をクリック | - | - |

---

## よくある質問（FAQ）

**Q1: [質問]**
A: [回答]

---

## 関連リンク

- 配信設定テンプレート: [リンク]
- 詳細マニュアル: [リンク]
```

### 3.3 ナレーション/画面操作の書き方

**ナレーション**:
- 敬語（です・ます調）で統一
- 1文は30秒以内で読める長さに
- 専門用語は初出時に簡単な説明を添える

**画面操作指示**:
- 操作対象を明確に記載（「左メニューの『配信管理』」など）
- クリック・入力・選択の区別を明記
- 入力する値がある場合は具体例を記載

## 4. 撮影・編集の技術仕様

### 4.1 推奨ツール

| 用途 | 推奨ツール | 備考 |
|------|-----------|------|
| 画面キャプチャ | OBS Studio（無料） | 画面録画+マイク音声同時収録 |
| 動画編集 | DaVinci Resolve（無料版） | 本格編集、テロップ、カラー調整 |
| 簡易確認 | Loom | ドラフト確認用 |

### 4.2 映像仕様

| 項目 | 仕様 |
|------|------|
| 解像度 | 1920x1080（Full HD） |
| フレームレート | 30fps |
| アスペクト比 | 16:9 |
| ビットレート | 8-12 Mbps |
| 出力形式 | MP4（H.264） |

### 4.3 音声仕様

| 項目 | 仕様 |
|------|------|
| サンプルレート | 48kHz |
| ビットレート | 192kbps以上 |
| 形式 | AAC |
| 音量 | -6dB〜-3dBを目安に正規化 |

### 4.4 テロップガイドライン

| 要素 | 仕様 |
|------|------|
| フォント | Noto Sans JP / ヒラギノ角ゴシック |
| サイズ | タイトル: 48-60pt、本文: 32-40pt |
| 配置 | 画面下部1/5のセーフエリア内 |
| 背景 | 半透明の黒または白 |
| 表示時間 | 最低3秒、長文は5秒以上 |
| 色 | 白文字+黒縁取り、強調時は黄色 |

### 4.5 BGMガイドライン

| 項目 | ガイドライン |
|------|-------------|
| 音量 | ナレーション音量の15-20%程度 |
| ジャンル | 落ち着いた企業系BGM |
| 利用素材 | 著作権フリー素材のみ |
| 推奨サイト | DOVA-SYNDROME、甘茶の音楽工房、YouTube Audio Library |
| 使用箇所 | オープニング・エンディングで使用 |

### 4.6 ファイル命名規則

| 種別 | 命名規則 | 例 |
|------|---------|-----|
| 台本 | `script-[番号]-[英字タイトル].md` | `script-01-admin-basic.md` |
| 完成動画 | `onboarding-video-[番号]-[英字タイトル]-v[バージョン].mp4` | `onboarding-video-01-admin-basic-v1.mp4` |
| サムネイル | `thumbnail-[番号]-[英字タイトル].png` | `thumbnail-01-admin-basic.png` |
| BGM | `bgm-[用途].mp3` | `bgm-opening.mp3` |
| RAW動画 | `raw-[番号]-[日付]-[テイク番号].mp4` | `raw-01-20250210-take1.mp4` |

## 6. 品質管理

### 6.1 レビュープロセス

```
鈴木（作成） → 嘉生（1次レビュー） → 修正 → 嘉生（最終確認） → 公開
```

### 6.2 修正フロー

| 修正レベル | 定義 | 対応 |
|-----------|------|------|
| 軽微 | テロップ誤字、BGM音量調整 | 即日修正、再レビュー不要 |
| 中程度 | 説明順序変更、一部再撮影 | 1-2日で修正、嘉生確認 |
| 重大 | 全体構成変更、全面再撮影 | スケジュール再調整 |

### 6.3 公開前チェックリスト

**内容チェック**:
- [ ] 操作手順に誤りがないか
- [ ] 用語が統一されているか
- [ ] 前提知識・事前視聴の案内があるか
- [ ] 次に視聴すべき動画の案内があるか

**技術チェック**:
- [ ] 音声が聞き取りやすいか
- [ ] 画面が見やすいか
- [ ] テロップに誤字脱字がないか
- [ ] BGMの音量は適切か

**メタ情報チェック**:
- [ ] 動画タイトルは正確か
- [ ] 説明文は適切か
- [ ] サムネイルは設定されているか
- [ ] 公開設定（限定公開）は正しいか
